@claude agent works without you

Claude that works without you. Mention @claude … on a GitHub issue and a session in Anthropic's cloud does the work and opens a PR — your laptop off, on your Max subscription. Two pieces, set up once.

  1. You mention @claude … in an issue — its body or a comment.
  2. A tiny dispatcher Action you own checks permissions and fires a cloud routine — it does zero Claude work itself.
  3. The routine clones the repo, implements the change, runs the tests, and opens a PR.
  4. From there, Auto-fix watches the PR and handles review comments and CI on its own.

1 · Set up the routine

One-time, in the web UI — the same for every repo:

  1. Install the Claude GitHub App on the repo — github.com/apps/claude (or accept the prompt during setup). Required: it gives the routine git access and receives the PR webhooks Auto-fix runs on.
  2. Open claude.ai/code/routines/new and point the routine at your repo.
  3. Paste the routine prompt as its instructions — .
  4. Choose API as the trigger.
  5. Under Behavior, turn on Auto-fix pull requests, then click Create.
  6. On creation the API token is generated automatically and shown once — copy it before you move to the routine overview, and save it as a repository secret (Settings → Secrets and variables → Actions → New repository secret) named CLAUDE_ROUTINE_TOKEN; you can't retrieve it afterward.
  7. Copy the trigger's fire URL (this one stays visible) and save it the same way as CLAUDE_ROUTINE_FIRE_URL.

2 · Add the dispatcher

Paste this into Claude Code in your repo — it writes the GitHub Action that detects @claude on an issue and fires your routine, using the two secrets you just created.
